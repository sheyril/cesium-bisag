(async) createJdbcCoverageFromRaster(layerPath, layerName, workspace, tileDir, levelsopt, srid, tileSize, interpolationopt) → {Promise.<{layerName: string, workspace: string, storeName: string}>}

Creates a JDBC ImageMosaic Coverage from a given Raster Layer. Takes care of creating the mapping file, following conventions, creating datastore and publishing the coverage to geoserver. For more information visit the official website and see tutorials on use of JDBC ImageMosaic
Parameters:
Name Type Attributes Default Description
layerPath string Path to the Raster Layer for which ImageMosaic is constructed
layerName string Name of the layer. This name is used to publish to geoserver
workspace string Name of the workspace in which layer has to be published
tileDir string Path to the tile Directory to use for creating tiles and pyramids. The directory is cleaned after use.
levels string | number <optional>
 2 Number of pyramid levels to be created
srid string SRS of the raster layer. (If wrong SRS given the layer preview would be blank on geoserver)
tileSize number Size of an individual tile in pixels
interpolation number <optional>
 1 interpolation to be used. Mostly the default value 1 is used. For custom interpolation visit the official website and see tutorials on use of JDBC ImageMosaic. If you are not sure let it go to default
Source:

Returns:
- Returns a promise that is resolved when the raster has been converted to JDBC ImageMosaic and published successfully
